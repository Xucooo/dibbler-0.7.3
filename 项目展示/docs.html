产品背景

传统网络采用端到端的通信,而在物联网环境中,通信方式往往与传统方式不一样。通信的目的往往是以信息为中心,因此,物联网设备需要考虑将 IPv6 地址与地理位置信息进行更深一步绑定，当需要访问某个地区的数据时,不需要考虑访问具体哪个设备的数据,而直接采用与地理位置对应的IPv6 地址或子网地址访问，以更直接的方式提供信息,获得更佳的效率。

产品设计

产品设计思路

通过考虑两种不同定位能力的节点（有自主定位功能和没有自主定位功能）在实际应用中的表现，完成地理位置与IPv6地址映射功能。

具体方案

1.IoT 设备位置信息与客户端的相关设计：完成 sink 节点获取链路信息,并将信息发送至地址配置服务器。主要分为两类：第一类,IoT 设备未配置定位系统,物联网设备无法自我定位,即需要通过与之连接的 sink 获取相关参数,计算出其地理位置并发给地址配置服务器端;第二类,设备在获取 IPv6 地址之前,采用 GPRS 或其它方式获取了地理信息,向服务器发送位置信息以及获取映射了的 IPv6 地址。

2.IPv6 地址与地理位置映射的算法研究：地址配置服务器构建基于地理位置的 IPv6 资源库;地址配置服务器服务器根据 sink与 IoT 设备之间的链路信息计算位置,并映射到资源库中的唯一 IPv6 地址。

3.定位配置和服务器端的相关设计：在现有的 DHCPv6 基础上完成与 sink、IoT 节点的通信和配置机制。地址配置服务器为 IoT 节点选择最佳 sink 节点;地址配置服务器通过 sink 为 IoT 设备配置 IPv6 地址。将地址池的 IPv6 按地理位置进行规划和排序。接收到物联网设备的位置信息后,与地址池的IPv6 地址进行映射。选择对应设备唯一的 IPv6 地址。并将确定的 IPv6地址分配给对应的物联网设备。

功能介绍

我们通过在服务器端和客户端分别装上我们所开发的软件，当用户移动到不同的AP内时，服务器根据用户所处的地理位置，采用DHCPv6协议为用户分配基于不同地理位置的IPv6地址。

创新点

1.通过IPv6地址能够有效确定设备的地理位置。

2.DHCP服务器提取报文中GPS信息，根据映射算法计算出与之对应的IPv6地址。

3.通过IPv6地址与地理位置信息绑定的机制能够有效提高物联网的使用效率。
